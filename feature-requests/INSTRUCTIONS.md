# TankFlare Feature Request Template Guide

If your feature request is approved, the developers may add it to the roadmap. The time frame will be decided based on how much work the feature will take to implement.

Please remember: **one feature per request**!

## Filename

The filename should be the name of the feature you want.

Examples:

```text
3D Aquascaping Tool.md
Oxygenation Checker.md
CO2 Calculator.md
```

## Annotated Template

If you do not know how to write in Markdown, check out this [cheatsheet](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax).

You can preview your Markdown by clicking the **Preview** button next to the **Edit** button.

```text
# Oxygenation Checker

## What do you want?

I want to be able to check if my tank has enough oxygenation for my fish. For example, I can enter whether my tank is open top, has a normal lid, or has a closed lid, since that affects gas exchange. I can also enter whether I have no airstone, a small airstone, a medium airstone, or a large airstone.

The checker should then tell me if my oxygenation setup is likely good enough for my fish, and how to improve it if not.

## Where do you want it in the UI?

In the checker UI, under the filter flow input area as an optional collapsible section. Since oxygenation is closely related to filter flow and surface agitation, it should appear near those settings instead of in a completely separate area.

The oxygenation result should also appear in the final checker results, with a short explanation and a few simple suggestions if the setup could be improved.
```

## Blank Template

Copy and paste this blank template:

```text
# [Feature Name]

## What do you want?

Be very descriptive and thorough so we can understand the exact feature you want. The more details, the better!

## Where do you want it in the UI?

Tell us exactly where you want the feature to appear so we can fit what you envisioned.
```

## Quick Checklist

- [ ] Filename is the feature name you want added
- [ ] Described your wanted feature in thorough detail
- [ ] Described where you want your feature with precision
